# Implementation Plan: Update Chat Help & Help Page for Comprehensive Chat Commands & Features

**Branch**: `046-chat-help-docs` | **Date**: 2026-03-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/046-chat-help-docs/spec.md`

**Note**: This template is filled in by the `/speckit.plan` command. See `.specify/templates/commands/plan.md` for the execution workflow.

## Summary

Expand the `/help` chat command output and the Help page to comprehensively document all chat capabilities beyond the existing six slash commands. The `/help` handler appends plain-text "Chat Features" and "Tips" sections to the dynamic command list. The Help page gains a new "Chat Features" card grid (seven cards using the existing `FeatureGuideCard` pattern), an enriched Slash Commands table with a dynamically-driven "Options" column sourced from `parameterSchema`, and two FAQ corrections (remove non-existent `/clear` reference; remove false "drag-and-drop" claim). All changes are frontend-only, modifying two source files (`help.ts`, `HelpPage.tsx`) plus adding `labels` to the language command's `parameterSchema` in `registry.ts`. Tests cover the expanded help output, new Chat Features section, enriched table, and corrected FAQ text.

### Phase 1 вЂ” Expand `/help` chat command output (US1)

**Files**: `solune/frontend/src/lib/commands/handlers/help.ts`

   - File Attachments: "Use the attachment button to add files (images, PDF, code, up to 10 MB)"
   - Voice Input: "Use the microphone button for speech-to-text (Chrome/Edge recommended)"
   - AI Enhance: "Toggle AI Enhance in the toolbar for smarter responses"
2. Append a "Tips" block after Chat Features with four lines:
   - "Type / to browse and autocomplete commands"
   - "You can also type help without the slash"
   - "Use Arrow Up/Down to browse previous messages"
   - "Visit the Help page for the full guide"
3. All output remains plain text вЂ” no Markdown syntax
4. Command list portion remains dynamically generated from `getAllCommands()`

### Phase 2 вЂ” Add "Chat Features" section to Help page (US2)

**Files**: `solune/frontend/src/pages/HelpPage.tsx`

1. Define a `CHAT_FEATURES` array with seven entries (title, description, icon, href)
2. Insert a new `<section>` between FAQ and Feature Guides using the existing `FeatureGuideCard` component
3. Grid layout: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` (7 cards reflow on mobile)
4. Cards: @Pipeline Mentions, File Attachments, Voice Input, AI Enhance, Task Proposals, Message History, Keyboard Shortcuts

### Phase 3 вЂ” Enrich Slash Commands table (US3)

**Files**: `solune/frontend/src/pages/HelpPage.tsx`, `solune/frontend/src/lib/commands/registry.ts`

1. Add `labels` to the language command's `parameterSchema` in `registry.ts`:
   ```ts
   parameterSchema: { type: 'enum', values: ['en', 'es', 'fr', 'de', 'ja', 'zh'], labels: { en: 'English', es: 'Spanish', fr: 'French', de: 'German', ja: 'Japanese', zh: 'Chinese' } }
   ```
2. Add an "Options" column to the commands table header
3. Add a helper function `getOptionsDisplay(cmd)` that reads `cmd.parameterSchema`:
   - If `type === 'enum'` with `labels`: show `"value (Label)"` comma-separated
   - If `type === 'enum'` without `labels`: show values comma-separated
   - If `passthrough`: show the syntax parameter format + "(admin only)"
   - Otherwise: show `"вЂ”"`
4. Add footer note below the table
