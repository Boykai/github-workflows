# Implementation Plan: Update Chat Help & Help Page for Comprehensive Chat Commands & Features

**Branch**: `046-chat-help-docs` | **Date**: 2026-03-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/046-chat-help-docs/spec.md`

**Note**: This template is filled in by the `/speckit.plan` command. See `.specify/templates/commands/plan.md` for the execution workflow.

## Summary

Expand the `/help` chat command output and the Help page to comprehensively document all six slash commands plus all non-command chat features (pipeline mentions, file attachments, voice input, AI enhance, task proposals, message history, and keyboard shortcuts). The approach is five-phased: (1) append Chat Features and Tips blocks to the `/help` handler plain-text output, (2) add a seven-card Chat Features section to HelpPage.tsx between FAQ and Feature Guides, (3) enrich the Slash Commands table with a dynamically-driven Options column sourced from `parameterSchema`, (4) fix two FAQ inaccuracies (`/clear` → `/agent`, "drag-and-drop" → attachment button), and (5) add/update unit and component tests for all changes. No new dependencies, no backend changes, no new components — all changes reuse existing patterns (FeatureGuideCard, plain-text formatting, registry-driven rendering).

## Implementation Phases

### Phase 1 — Expand `/help` Command Output (User Story 1, P1)

**File**: `solune/frontend/src/lib/commands/handlers/help.ts`

1. After the dynamically-generated command list, append a blank line separator
2. Append a "Chat Features:" header followed by four feature lines:
   - `@Pipeline Mentions — Type @ to invoke a pipeline by name`
   - `File Attachments — Use the attachment button to add files (images, PDF, code, up to 10 MB)`
   - `Voice Input — Use the microphone button for speech-to-text (Chrome/Edge recommended)`
   - `AI Enhance — Toggle AI Enhance in the toolbar for smarter responses`
3. Append another blank line separator
4. Append a "Tips:" header followed by four tip lines:
   - `Type / to browse and autocomplete commands`
   - `You can also type help without the slash`
   - `Use Arrow Up/Down to browse previous messages`
   - `Visit the Help page for the full guide`
5. All output remains plain text (no Markdown)
6. The command list portion remains dynamically generated from registry

### Phase 2 — Add Chat Features Section to Help Page (User Story 2, P2)

**File**: `solune/frontend/src/pages/HelpPage.tsx`

1. Define a `CHAT_FEATURES` array of 7 feature objects with `title`, `description`, `icon` (from lucide-react)
2. Insert a new `<section>` with heading "Chat Features" between the FAQ section and the Feature Guides section
3. Render 7 feature cards in a responsive grid using a card layout consistent with `FeatureGuideCard` styling (but without navigation links since these are informational, not page links)
4. Cards: @Pipeline Mentions, File Attachments, Voice Input, AI Enhance, Task Proposals, Message History, Keyboard Shortcuts
5. Grid responsive: 1 column on mobile, 2 on sm, 3-4 on lg

### Phase 3 — Enrich Slash Commands Table (User Story 3, P3)

**File**: `solune/frontend/src/pages/HelpPage.tsx`

1. Add a 4th column "Options" to the commands table header
2. For each command row, render the Options cell:
   - If `cmd.parameterSchema?.type === 'enum'` and `cmd.parameterSchema.labels`: show labeled values (e.g., "en (English)")
   - If `cmd.parameterSchema?.type === 'enum'` (no labels): show values joined by ", "
   - If `cmd.passthrough`: show the syntax parameters + "(admin only)" badge
   - Otherwise: show "—"
3. Add a footer note below the table: `Type / in chat to autocomplete commands. You can also type help without the slash.`

### Phase 4 — Fix FAQ Inaccuracies (User Story 4, P4)

**File**: `solune/frontend/src/pages/HelpPage.tsx`

1. `chat-voice-1`: Replace `/clear` with `/agent` in the answer text; update the command list to be accurate
2. `chat-voice-2`: Remove "drag-and-drop" claim; replace with "click the attachment button in the chat toolbar"

### Phase 5 — Tests (User Story 5, P5)

**Files**: `help.test.ts`, `HelpPage.test.tsx`

1. Update `help.test.ts`:
   - Add test: output includes "Chat Features" section content
   - Add test: output includes "Tips" section content
   - Add test: output includes specific feature lines (@Pipeline Mentions, File Attachments, Voice Input, AI Enhance)
   - Add test: output includes specific tip lines (autocomplete, help alias, history, Help page)
2. Create `HelpPage.test.tsx`:
   - Test: Chat Features section renders with 7 cards
   - Test: Each card has correct title
   - Test: Commands table has Options column
   - Test: Options column shows correct values for each command
   - Test: FAQ `chat-voice-1` does not contain `/clear`
   - Test: FAQ `chat-voice-2` does not contain "drag-and-drop"
   - Test: Footer note is present below commands table
