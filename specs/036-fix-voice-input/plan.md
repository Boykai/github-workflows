# Implementation Plan: Fix Chat Microphone Voice Input ‚Äî Incorrect Browser Support Detection

**Branch**: `036-fix-voice-input` | **Date**: 2026-03-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/036-fix-voice-input/spec.md`

## Summary

Fix the chat microphone button that incorrectly reports "Voice input not support in this browser" in Firefox and other supported browsers. The root cause is an overly narrow browser-support check that only tests for the vendor-prefixed `webkitSpeechRecognition` constructor, missing Firefox's unprefixed `SpeechRecognition` (available since Firefox 85+). The fix expands the detection function to check both `window.SpeechRecognition` and `window.webkitSpeechRecognition`, corrects the grammatical error in the error message ("not support" ‚Üí "not supported"), adds a `useVoiceInput` hook encapsulating Web Speech API integration with permission handling and interim transcription, and provides a `VoiceInputButton` component with visual recording state (pulsing animation). All changes are frontend-only, confined to 3 new files and minor integration wiring in existing chat components.
