## Summary

Refactor the Settings page from a monolithic collapsible form (18 components, 5 sections) into a clean 4-tab layout (Essential / Secrets / Preferences / Admin) using Shadcn Tabs. Add a new GitHub Repository Environment Secrets feature for MCP API keys using `githubkit` environment secrets API with NaCl sealed-box encryption via `pynacl`. Consolidate redundant wrapper components, preserve leaf components and per-section save buttons, and integrate secret status warnings into MCP presets on the Tools page.

**Approach**: Phase-ordered implementation — Phase 1 (backend secrets service) enables Phase 3 (secrets UI). Phase 2 (tab layout) enables Phases 3–4. Phase 5 (cleanup & polish) depends on all prior phases.

## Implementation Phases

### Phase 1: Backend — GitHub Environment Secrets Service *(blocks Phase 3)*

#### Task 1.1: Add `pynacl` Dependency

**FR**: FR-016 | **File**: `solune/backend/pyproject.toml`
**Change**: Add `pynacl>=1.5.0` to the `dependencies` list. PyNaCl provides the NaCl sealed-box encryption required by GitHub's environment secrets API.

#### Task 1.2: Create Secrets Service

**FR**: FR-015, FR-016, FR-017, FR-018 | **New file**: `solune/backend/src/services/secrets_service.py`
**Change**: Create a new service module with four async functions:
- `list_secrets(access_token, owner, repo, environment_name)` → calls `client.rest.actions.async_list_environment_secrets()`, returns list of `{name, created_at, updated_at}`
- `set_secret(access_token, owner, repo, environment_name, secret_name, secret_value)` → fetches public key via `async_get_environment_public_key()`, encrypts value with `nacl.public.SealedBox`, calls `async_create_or_update_environment_secret()`
- `delete_secret(access_token, owner, repo, environment_name, secret_name)` → calls `async_delete_environment_secret()`
- `get_or_create_environment(access_token, owner, repo, environment_name)` → calls `async_create_or_update_environment()` to ensure the environment exists before secret operations
**Pattern**: Import `GitHubClientFactory` from `services/github_projects`, use `factory.get_client(access_token)` for all API calls. Follow existing service function signature pattern (access_token as first parameter).

#### Task 1.3: Create Secrets API Router

**FR**: FR-015, FR-016, FR-017, FR-019, FR-020, FR-021, FR-029 | **New file**: `solune/backend/src/api/secrets.py`
**Change**: Create a FastAPI router with endpoints:
- `GET /{owner}/{repo}/{environment}` → list secret names + metadata (never returns values)
- `PUT /{owner}/{repo}/{environment}/{secret_name}` → create/update (body: `{value: str}`)
- `DELETE /{owner}/{repo}/{environment}/{secret_name}` → remove
- `GET /{owner}/{repo}/{environment}/check` → bulk check (query param: `names=KEY1,KEY2`; returns `{KEY1: true, KEY2: false}`)
**Auth**: Reuse `require_session` dependency from `api/dependencies.py`.
**Validation**: Pydantic path parameter validator for `secret_name` (`^[A-Z][A-Z0-9_]*$`, max 255 chars). Request body validator for `value` (max 64KB). Auto-call `get_or_create_environment()` before set operations.

#### Task 1.4: Register Secrets Router

**FR**: (infrastructure) | **File**: `solune/backend/src/api/__init__.py`
**Change**: Add `from .secrets import router as secrets_router` and `router.include_router(secrets_router, prefix="/secrets", tags=["secrets"])` following the existing pattern (reference: settings_router, signal_router registrations).

#### Task 1.5: Add `required_secrets` to MCP Preset Model

**FR**: FR-028 | **Files**: `solune/backend/src/models/tools.py`, `solune/backend/src/services/tools/presets.py`
**Change**: Add `required_secrets: list[str] = []` field to `McpPresetResponse` in `models/tools.py`. In `presets.py`, update the Context7 preset to include `required_secrets=["COPILOT_MCP_CONTEXT7_API_KEY"]`. All other presets default to empty list (backward-compatible).
