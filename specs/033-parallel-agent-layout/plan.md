# Implementation Plan: Parallel Agent Layout in Pipelines

**Branch**: `033-parallel-agent-layout` | **Date**: 2026-03-10 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/033-parallel-agent-layout/spec.md`

## Summary

Enable pipeline builders to arrange Agents side-by-side within a single pipeline stage so those Agents execute in parallel, all completing before the pipeline advances to the next sequential stage. Currently the pipeline canvas only supports stacked (series) Agent execution. This feature extends the data model with a per-stage `execution_mode` field (`"sequential"` | `"parallel"`), updates the pipeline execution engine to fan out parallel-stage Agents via `asyncio.gather`, and enhances the frontend canvas to render parallel Agents horizontally in a visually grouped container with distinct iconography and connector behavior.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript 5.x (frontend)
**Primary Dependencies**: FastAPI, Pydantic, aiosqlite (backend); React 18, TanStack React Query, @dnd-kit/core + @dnd-kit/sortable (frontend); Tailwind CSS, shadcn/ui (styling)
**Storage**: SQLite via aiosqlite — `pipeline_configs.stages` stored as JSON text column
**Testing**: pytest + pytest-asyncio (backend); Vitest + happy-dom (frontend)
**Target Platform**: Linux server (backend), modern browsers 1280px+ (frontend)
**Project Type**: Web application (backend + frontend)
**Performance Goals**: Parallel stage Agents must all start within 2 seconds of each other (SC-002); canvas remains responsive and readable with up to 4 parallel Agents per stage at 1280px viewport width (SC-007)
**Constraints**: Backward-compatible — existing sequential pipelines must work unchanged without migration (FR-008); API contracts versioned or additive-only to avoid breaking clients
**Scale/Scope**: Pipeline canvases typically contain 3–7 stages; parallel stages expected to hold 2–4 Agents; ~50 concurrent users
