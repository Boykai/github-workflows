# Data Model: Codebase Audit & Refactor

**Feature**: 018-codebase-audit-refactor
**Date**: 2026-03-05

## New Entities

### ChatMessage (SQLite table: `chat_messages`)

Persists chat messages that were previously stored in-memory.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| message_id | TEXT | PRIMARY KEY | UUID generated by caller |
| session_id | TEXT | NOT NULL, INDEXED | Groups messages by user session |
| sender_type | TEXT | NOT NULL, CHECK IN ('user', 'assistant', 'system') | Who sent the message |
| content | TEXT | NOT NULL | Message body |
| action_type | TEXT | NULLABLE | Optional action identifier (e.g., 'create_task') |
| action_data | TEXT | NULLABLE | JSON-encoded action payload |
| timestamp | TEXT | NOT NULL, DEFAULT now | ISO 8601 UTC timestamp |

**Relationships**: None (messages are standalone within a session)
**Validation**: `sender_type` must be one of the three allowed values
**State transitions**: N/A (messages are immutable once created)

### ChatProposal (SQLite table: `chat_proposals`)

Persists AI task proposals that were previously stored in-memory.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| proposal_id | TEXT | PRIMARY KEY | UUID generated by caller |
| session_id | TEXT | NOT NULL, INDEXED | Links proposal to a session |
| original_input | TEXT | NOT NULL | User's original request text |
| proposed_title | TEXT | NOT NULL | AI-generated task title |
| proposed_description | TEXT | NOT NULL | AI-generated task description |
| status | TEXT | NOT NULL, CHECK IN ('pending', 'confirmed', 'edited', 'cancelled'), DEFAULT 'pending' | Current state |
| edited_title | TEXT | NULLABLE | User-edited title (if edited) |
| edited_description | TEXT | NULLABLE | User-edited description (if edited) |
| created_at | TEXT | NOT NULL, DEFAULT now | ISO 8601 UTC |
| expires_at | TEXT | NULLABLE | Expiration time for pending proposals |

**Relationships**: Belongs to a session (via session_id)
**Validation**: `status` must be one of the four allowed values
**State transitions**: `pending` → `confirmed` | `edited` | `cancelled` (terminal states)

### ChatRecommendation (SQLite table: `chat_recommendations`)

Persists issue recommendations that were previously stored in-memory.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| recommendation_id | TEXT | PRIMARY KEY | UUID generated by caller |
| session_id | TEXT | NOT NULL, INDEXED | Links to a session |
| data | TEXT | NOT NULL | JSON-encoded recommendation payload (IssueRecommendation model serialized) |
| status | TEXT | NOT NULL, CHECK IN ('pending', 'accepted', 'rejected'), DEFAULT 'pending' | Current state |
| created_at | TEXT | NOT NULL, DEFAULT now | ISO 8601 UTC |

**Relationships**: Belongs to a session (via session_id)
**Validation**: `status` must be one of the three allowed values
**State transitions**: `pending` → `accepted` | `rejected` (terminal states)

## Modified Entities

### CopilotClientPool (new class in `completion_providers.py`)

In-memory bounded cache of CopilotClient instances. Not a database entity.

| Field | Type | Notes |
|-------|------|-------|
| _clients | BoundedDict[str, Any] | Max 50 entries, keyed by token hash |
| _lock | asyncio.Lock | Prevents concurrent creation of same client |

**Methods**:
- `get_or_create(token: str) -> CopilotClient` — returns cached or creates new client
- `cleanup() -> None` — stops all cached clients, clears pool
- `remove(token: str) -> None` — stops and removes a single client

### RateLimitState (enhancement to existing `_last_rate_limit` in service.py)

| Field | Type | Notes |
|-------|------|-------|
| remaining | int | Calls remaining in current window |
| reset_at | float | Unix timestamp when limit resets |
| _last_request_time | float | Monotonic clock of last request (for inter-call spacing) |
| _min_request_interval | float | Minimum seconds between calls (default 0.5) |

## Unchanged Entities

The following existing entities are not modified by this feature:
- All existing SQLite tables (users, projects, tasks, settings, etc.)
- All existing Pydantic models (ChatMessage, AITaskProposal, IssueRecommendation — shapes unchanged, only storage backend changes)
- Frontend data models (no schema changes, backward-compatible API)
