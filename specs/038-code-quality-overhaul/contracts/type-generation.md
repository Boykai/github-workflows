# Contract: Type Generation Pipeline

**Feature**: 038-code-quality-overhaul | **Status**: Draft

## Purpose

Define the contract for generating TypeScript types from the backend's OpenAPI schema and Python constants. This eliminates manually maintained `types/index.ts` definitions and ensures frontend/backend type alignment is enforced at build time.

## Pipeline Overview

```
┌─────────────────────┐     curl      ┌──────────────────────┐
│ FastAPI Backend      │─────────────>│ /openapi.json         │
│ (Pydantic models)   │              │ (auto-generated)      │
└─────────────────────┘              └──────────┬───────────┘
                                                │
                                    openapi-typescript
                                                │
                                                ▼
                                    ┌──────────────────────┐
                                    │ types/generated.ts    │
                                    │ (TypeScript interfaces)│
                                    └──────────────────────┘

┌─────────────────────┐   Python     ┌──────────────────────┐
│ backend/src/         │   script    │ constants/generated.ts │
│ constants.py         │────────────>│ (TS const exports)    │
└─────────────────────┘              └──────────────────────┘
```

## OpenAPI → TypeScript Types

### Input

- **Source**: `GET http://localhost:8000/openapi.json`
- **Format**: OpenAPI 3.1.x JSON (auto-generated by FastAPI from Pydantic models)
- **Scope**: All response models, request bodies, and query parameter schemas

### Output

- **Destination**: `frontend/src/types/generated.ts`
- **Format**: TypeScript interface declarations (read-only, auto-generated)
- **Tool**: `openapi-typescript` (npm package)

### Generation Command

```bash
#!/usr/bin/env bash
# scripts/generate-types.sh

set -euo pipefail

BACKEND_URL="${BACKEND_URL:-http://localhost:8000}"
FRONTEND_DIR="frontend/src"

# Step 1: Generate TypeScript types from OpenAPI schema
npx openapi-typescript "${BACKEND_URL}/openapi.json" \
  --output "${FRONTEND_DIR}/types/generated.ts"

# Step 2: Generate constants from Python source
python scripts/generate-constants.py \
  --input backend/src/constants.py \
  --output "${FRONTEND_DIR}/constants/generated.ts"

echo "Type generation complete."
```

### Generated File Header

All generated files include a header to prevent manual editing:

```typescript
/* eslint-disable */
/* This file is auto-generated. Do not edit manually. */
/* Run: npm run generate:types */
```

## Constants Generation

### Input

- **Source**: `backend/src/constants.py`
- **Target exports**: Top-level `str`, `int`, `float`, `bool` constants and string literal collections

### Output

- **Destination**: `frontend/src/constants/generated.ts`
- **Format**: TypeScript `export const` declarations

### Mapping Rules

| Python Type | TypeScript Output |
|-------------|-------------------|
| `str` constant | `export const FOO = "bar" as const;` |
| `int` constant | `export const MAX = 100 as const;` |
| `list[str]` constant | `export const ITEMS = ["a", "b"] as const;` |
| `dict` constant | `export const MAP = { ... } as const;` |
| Class/function | Skipped (not a constant) |

### Generation Script

```python
#!/usr/bin/env python3
"""scripts/generate-constants.py — Extract constants from Python to TypeScript."""

import ast
import argparse
import json
from pathlib import Path


def extract_constants(source: str) -> list[tuple[str, str]]:
    """Parse Python source and extract top-level constant assignments."""
    tree = ast.parse(source)
    constants = []
    for node in ast.iter_child_nodes(tree):
        if isinstance(node, ast.Assign) and len(node.targets) == 1:
            target = node.targets[0]
            if isinstance(target, ast.Name) and target.id.isupper():
                value = ast.literal_eval(node.value)
                constants.append((target.id, json.dumps(value)))
    return constants


def generate_typescript(constants: list[tuple[str, str]]) -> str:
    """Generate TypeScript const declarations."""
    lines = [
        "/* eslint-disable */",
        "/* This file is auto-generated from backend/src/constants.py */",
        "/* Run: npm run generate:types */",
        "",
    ]
    for name, value in constants:
        lines.append(f"export const {name} = {value} as const;")
    return "\n".join(lines) + "\n"


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--input", required=True)
    parser.add_argument("--output", required=True)
    args = parser.parse_args()

    source = Path(args.input).read_text()
    constants = extract_constants(source)
    typescript = generate_typescript(constants)
    Path(args.output).write_text(typescript)


if __name__ == "__main__":
    main()
```

## Integration Points

### npm Scripts (package.json additions)

```json
{
  "scripts": {
    "generate:types": "bash ../scripts/generate-types.sh",
    "prebuild": "npm run generate:types"
  }
}
```

### Pre-commit Hook

The existing `scripts/pre-commit` hook should add a stale-check:

```bash
# Check if generated types are stale
if git diff --cached --name-only | grep -q "backend/src/.*\\.py"; then
  npm run --prefix frontend generate:types
  git add frontend/src/types/generated.ts frontend/src/constants/generated.ts
fi
```

### CI Validation

Add to GitHub Actions workflow:

```yaml
- name: Validate generated types are fresh
  run: |
    npm run --prefix frontend generate:types
    git diff --exit-code frontend/src/types/generated.ts frontend/src/constants/generated.ts
```

## Migration Strategy

1. **Phase A**: Generate `types/generated.ts` alongside existing `types/index.ts`
2. **Phase B**: Update imports incrementally to use generated types where available
3. **Phase C**: Delete manual type definitions that are fully covered by generated types
4. **Phase D**: Keep `types/index.ts` for frontend-only types (form state, UI state) not in API schema
