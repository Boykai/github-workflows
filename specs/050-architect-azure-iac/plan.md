# Implementation Plan: Architect Agent for Azure IaC

**Branch**: `050-architect-azure-iac` | **Date**: 2026-03-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/050-architect-azure-iac/spec.md`

## Summary

Create a new Architect utility agent (`.github/agents/architect.agent.md`) that generates Infrastructure as Code using Bicep, Azure Developer CLI (`azd`), and architecture diagrams. Configure Azure MCP in both local and remote MCP config files. Extend the Solune "Create New App" flow to accept optional Azure credentials (Client ID and Client Secret), storing them as encrypted GitHub repository secrets via the GitHub Secrets API. The Architect agent is always invoked after new app scaffolding to generate `infra/`, `azure.yaml`, Mermaid architecture diagrams, and a "Deploy to Azure" button in the app README. Research confirms: Azure MCP GA 1.0 via `npx @azure/mcp@latest`, Bicep MCP auto-provided by VS Code extension, `pynacl` for GitHub Secrets encryption, and existing Mermaid `.mmd` conventions for diagrams.

## Technical Context

**Language/Version**: Python 3.13 (backend), TypeScript 5.9 (frontend), Bicep (IaC — generated by agent, not compiled at dev time)
**Primary Dependencies**: FastAPI + Pydantic + GitHubKit + PyNaCl (backend), React 19 + Vite 7 + TanStack Query (frontend), Azure MCP + Context7 + CodeGraphContext (agent tooling)
**Storage**: aiosqlite (SQLite — no schema changes; Azure credentials stored in GitHub Secrets, not local DB)
**Testing**: pytest + pytest-asyncio (backend), Vitest + Testing Library (frontend)
**Target Platform**: Linux (Docker containers), VS Code Agent mode, GitHub Custom Agents (remote)
**Project Type**: Web application (backend + frontend monorepo under `solune/`) + Agent definition (`.github/agents/`)
**Performance Goals**: Azure credential storage completes in <5s (2 GitHub API calls per secret); agent invocation is async and does not block app creation response
**Constraints**: GitHub Secrets API requires `libsodium` encryption (`pynacl` dependency); `repo` OAuth scope must include secrets access; Azure MCP requires Node.js for `npx`
**Scale/Scope**: 1 new agent file, 1 new prompt file, 2 MCP config edits, 1 doc update, 2 backend model fields, 1 new service method, 1 service method modification, 2 frontend type fields, ~10 lines of state/UI in AppsPage.tsx
