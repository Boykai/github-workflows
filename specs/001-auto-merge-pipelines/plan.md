# Implementation Plan: Auto Merge — Automatically Squash-Merge Parent PRs When Pipelines Complete

**Branch**: `001-auto-merge-pipelines` | **Date**: 2026-03-22 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-auto-merge-pipelines/spec.md`

## Summary

Add automatic squash-merge capability to the pipeline orchestrator so that parent PRs are merged when all pipeline steps complete successfully, without requiring human intervention. The feature introduces a boolean `auto_merge` flag at both project and pipeline levels (OR logic — either true activates), skips the human agent step when active, implements a new `_attempt_auto_merge()` function for merge orchestration, adds a DevOps repository agent for CI failure recovery (capped at 2 retries), and provides real-time WebSocket notifications for merge events. The existing queue-mode pattern (settings store, `__workflow__` canonical row, in-memory cache, UI toggle) serves as the blueprint for the data model and persistence layer.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript / React (frontend)
**Primary Dependencies**: FastAPI, Pydantic, aiosqlite (backend); React, TanStack Query, lucide-react, sonner (frontend)
**Storage**: SQLite via aiosqlite — project_settings and pipeline_configs tables; in-memory L1 caches with write-behind persistence
**Testing**: pytest (backend), Vitest with happy-dom (frontend)
**Target Platform**: Linux server (Docker), modern web browser
**Project Type**: Web application (backend + frontend monorepo under `solune/`)
**Performance Goals**: Toggle enablement < 10 seconds (SC-001); merge execution < 60 seconds after pipeline completion (SC-002); real-time notifications < 5 seconds (SC-006)
**Constraints**: SQLite single-writer; eventual consistency via 10-second TTL caches; atomic pipeline transitions (merge before external effects)
**Scale/Scope**: Per-project setting; concurrent pipelines per project governed by queue mode; DevOps retry cap of 2 per pipeline
