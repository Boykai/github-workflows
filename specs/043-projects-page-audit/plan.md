## Summary

Comprehensive audit and refactoring of the Projects page (`ProjectsPage.tsx`, 630 lines) to meet modern best-practice standards across 10 audit dimensions: component architecture, data fetching, loading/error/empty states, type safety, accessibility, text/copy polish, styling, performance, test coverage, and code hygiene. The page will be decomposed from a single 630-line monolith into a ≤250-line orchestrator delegating to focused sub-components in `src/components/board/`. Complex inline state logic will be extracted into existing hooks (`useProjectBoard`, `useProjects`, `useBoardControls`, `useBoardRefresh`). All audit findings will be addressed across six implementation phases: discovery, structural fixes, states & error handling, accessibility & UX polish, testing, and validation.
