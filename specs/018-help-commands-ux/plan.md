# Implementation Plan: Enhance #help and General # Commands with Robust, Best-Practice Chat UX

## Summary

Enhance the existing chat command system to provide categorized `#help` output, context-sensitive error guidance with "Did you mean?" suggestions, consistent response formatting, and per-command help via `#help <command>`. All changes are frontend-only, extending the existing `CommandDefinition` type with an optional `category` field and refactoring the help handler to support categorized display, single-command lookup, and fuzzy matching for unrecognized commands.

## Technical Context

**Language/Version**: TypeScript 5.x (ES2022 target)
**Primary Dependencies**: React 18, Vite, @tanstack/react-query, Radix UI
**Storage**: N/A (client-side command registry only — no persistence needed)
**Testing**: Vitest with happy-dom environment, @testing-library/react
**Target Platform**: Web browser (desktop + mobile, 320px minimum viewport)
**Project Type**: Web application (frontend + backend monorepo)
**Performance Goals**: Instant command response (<50ms) — all processing is local
**Constraints**: Plain-text rendering (no Markdown parser in chat); responses must be readable at 320px width without horizontal scrolling
**Scale/Scope**: 6 existing commands, expected to grow to ~20; single command registry in frontend
