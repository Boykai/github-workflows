# Implementation Plan: Phase 3 — Testing: Coverage, Mutation Enforcement, E2E, Property-Based & Keyboard Navigation

**Branch**: `001-phase-3-testing` | **Date**: 2026-03-22 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-phase-3-testing/spec.md`

## Summary

Implement a fully enforced, multi-layered test suite across the Solune platform: raise frontend Vitest coverage thresholds in two incremental phases (50→65→75%), extend backend per-file coverage floors (board.py ≥80%, pipelines.py ≥80%, pipeline.py ≥85%, agent_creator.py ≥70%), add a full-workflow integration test via httpx.ASGITransport, extend FIFO queue integration tests, make mutation testing CI-blocking (Stryker break=50, mutmut kill ratio ≥60%), add property-based queue tests with Hypothesis RuleBasedStateMachine (≥200 examples), and create a keyboard navigation E2E suite with axe-core accessibility scans. All work uses existing tooling — no new dependencies required.

## Implementation Phases

### Phase A — Frontend Coverage (P1) — Parallel with Phase B

**Goal**: Raise Vitest coverage from 50/44/41/50 to 75/65/65/75 in two incremental PRs.

**Sub-steps**:

1. **A.1 — Hook tests** (highest ROI): Write tests for the 6 untested hooks using the TanStack Query wrapper + `createMockApi()` pattern:
   - `useActivityFeed.test.tsx`
   - `useBoardDragDrop.test.tsx`
   - `useCommandPalette.test.tsx`
   - `useEntityHistory.test.tsx`
   - `useInfiniteList.test.tsx`
   - `useUnsavedPipelineGuard.test.tsx`

2. **A.2 — High-LOC component tests**: Write tests for untested components:
   - `ProjectBoard.test.tsx`
   - `ChatInterface.test.tsx`
   - `CleanUpConfirmModal.test.tsx`
   - `PipelineAnalytics.test.tsx`
   - `MarkdownRenderer.test.tsx`
   - `McpSettings.test.tsx`
   - `WorkflowSettings.test.tsx`

3. **A.3 — Phase 1 threshold raise**: Update `vitest.config.ts` thresholds to 65/55/55/65, verify `npm run test:coverage` passes.

4. **A.4 — Phase 2 threshold raise**: Update `vitest.config.ts` thresholds to 75/65/65/75, verify `npm run test:coverage` passes.

**Verification**: `npm run test:coverage` passes at each threshold level.

### Phase B — Backend Coverage (P1) — Parallel with Phase A

**Goal**: Meet per-file coverage floors across 4 critical backend modules.

**Sub-steps**:

1. **B.1 — board.py ≥80%**: Extend `test_api_board.py` with:
   - Column transform edge cases
   - Rate-limit recovery (429 response handling)
   - Token expiration and refresh flows
   - Cache hash computation branches
   - Error branches (116 missing statements)

2. **B.2 — pipelines.py ≥80%**: Extend pipeline API tests with:
   - Queue mode routing (L388-406)
   - Position calculation logic
   - Dequeue trigger paths
   - Sub-issue error handling (108 missing statements)

3. **B.3 — pipeline.py ≥85%**: Extend copilot polling pipeline tests with:
   - `_dequeue_next_pipeline()` complete path coverage
   - Grace period expiration and stale reclaim
   - BoundedDict race conditions (230 missing statements)

4. **B.4 — agent_creator.py ≥70%**: Extend agent creator tests with:
   - Exception paths (GitHub API errors, config validation)
   - Config parsing edge cases (malformed YAML, missing fields)
   - Tool assignment logic branches (246 missing statements)

**Verification**: `pytest --cov=src/api/board`, `pytest --cov=src/api/pipelines`, `pytest --cov=src/services/copilot_polling/pipeline`, `pytest --cov=src/services/agent_creator` each meet floors.

### Phase C — Full-Workflow Integration Test (P1) — Depends on Phase B

**Goal**: Implement end-to-end pipeline lifecycle integration test.

**Sub-steps**:

1. **C.1 — Test scaffold**: Create `tests/integration/test_full_workflow.py` using the `httpx.ASGITransport` pattern from `test_webhook_dispatch.py`:
   - Build FastAPI app with all routers
   - Configure HMAC webhook signing
   - Set up test fixtures (mock settings, services, sessions)

2. **C.2 — Issue lifecycle**: Test issue creation → project add → pipeline launch → agent assignment.

3. **C.3 — Status transitions**: Test Backlog → Ready → In Progress → In Review progression.

4. **C.4 — PR lifecycle**: Test PR creation webhook → pipeline state update → PR merge webhook → cleanup trigger → dequeue next pipeline.

**Verification**: `pytest tests/integration/test_full_workflow.py -v` — pipeline traverses all 4 statuses.

### Phase D — Mutation Testing Enforcement (P2) — Parallel with E, F

**Goal**: Make mutation testing CI jobs blocking instead of informational.

**Sub-steps**:

1. **D.1 — Stryker threshold**: In `stryker.config.mjs`, change `break: null` to `break: 50`.

2. **D.2 — Stryker CI**: In `mutation-testing.yml`, remove `continue-on-error: true` from the `frontend-mutation` job's Stryker step.

3. **D.3 — mutmut aggregation**: Add a new `backend-mutation-check` job to `mutation-testing.yml` that:
   - Depends on all `backend-mutation` matrix shards
   - Downloads all mutmut report artifacts
   - Parses kill ratios from each shard report
   - Fails the workflow if any shard kill ratio < 60%

4. **D.4 — mutmut CI**: Remove `continue-on-error: true` from the `backend-mutation` job's mutmut run step.

**Verification**: `npx stryker run` exits non-zero when score < 50; CI mutation jobs fail on low kill ratios.

### Phase E — Property-Based Queue Tests (P2) — Parallel with D, F

**Goal**: Extend property-based testing for queue operations with ≥200 Hypothesis examples.

**Sub-steps**:

1. **E.1 — State machine queue rules**: Extend `test_pipeline_state_machine.py` with:
   - `enqueue_pipeline` rule (idle → queued when active pipeline exists)
   - `dequeue_pipeline` rule (queued → running when no active pipeline)
   - `cancel_queued` rule (queued → cancelled)

2. **E.2 — State machine invariants**: Add invariants:
   - FIFO order preserved (queued pipelines sorted by started_at)
   - Queued pipelines have no assigned agent
   - Active pipeline count per project never exceeds 1

3. **E.3 — Blocking queue property tests**: Create `tests/property/test_blocking_queue.py`:
   - Exercise `count_active_pipelines_for_project()` with random state combinations
   - Exercise `get_queued_pipelines_for_project()` FIFO ordering invariant
   - Test `should_skip_agent_trigger()` exclusivity within grace period
   - Test stale reclaim at > 120 seconds
   - Run across ≥ 200 Hypothesis examples

**Verification**: `pytest tests/property/ -v` — all queue rules pass across 200 examples.

### Phase F — Keyboard Navigation E2E & Accessibility (P2) — Parallel with D, E

**Goal**: Create keyboard navigation suite and extend accessibility coverage.

**Sub-steps**:

1. **F.1 — Keyboard navigation suite**: Create `e2e/keyboard-navigation.spec.ts`:
   - Per-page Tab order verification
   - Enter/Space activation assertions
   - Escape closing modals
   - Focus trapping in modal dialogs
   - Correct initial focus on page load

2. **F.2 — Focus assertions in existing specs**: Extend with `expect(locator).toBeFocused()`:
   - `board-navigation.spec.ts` — Tab/Enter/Escape flows
   - `agent-creation.spec.ts` — Tab/Enter/Escape flows
   - `chat-interaction.spec.ts` — Tab/Enter/Escape flows

3. **F.3 — axe-core scan extension**: Add `AxeBuilder` accessibility scans to:
   - `agent-creation.spec.ts` (not present)
   - `pipeline-monitoring.spec.ts` (not present)
   - `mcp-tool-config.spec.ts` (not present)
   - `chat-interaction.spec.ts` (not present)
   - `keyboard-navigation.spec.ts` (new file)

4. **F.4 — Cross-service E2E evaluation**: Document feasibility assessment for docker-compose + DB + GitHub API mock + auth bypass as future work in research.md.

**Verification**: `npx playwright test keyboard-navigation.spec.ts` passes; axe-core scans present in ≥8 specs.

### Phase G — FIFO Queue Integration (P2) — After Phase B stabilizes

**Goal**: Extend queue integration tests for strict 3+ pipeline FIFO ordering.

**Sub-steps**:

1. **G.1 — Two-pipeline FIFO**: Pipeline A launches → Pipeline B queued → A completes → B dequeues automatically.

2. **G.2 — Three-pipeline strict ordering**: Pipelines A, B, C submitted in order → A runs, B and C queued → A completes → B starts → B completes → C starts. Assert strict FIFO order throughout.

**Verification**: Extended `test_queue_mode.py` passes with 3+ pipeline FIFO assertions.
