# Implementation Plan: Deep UI/UX Tooltip & Hover Coverage

**Branch**: `002-tooltip-hover-coverage` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-tooltip-hover-coverage/spec.md`

**Note**: This template is filled in by the `/speckit.plan` command. See `.specify/templates/commands/plan.md` for the execution workflow.

## Summary

Deliver full-coverage tooltip and hover UX across all 8 pages and all component groups in the Solune frontend. The approach is three-tiered: (1) install missing Radix UI primitives (`@radix-ui/react-hover-card`, `@radix-ui/react-popover`) and create reusable wrapper components modeled on the existing `tooltip.tsx`, (2) fill every identified tooltip/hover gap per component area by extending the centralized `tooltip-content.ts` registry and adding hover cards for rich entity previews, and (3) migrate 3+ manually built popover patterns to the standardized Radix popover, validate hover styling consistency, and audit for accessibility and motion compliance. Research confirmed all technical decisions align with the existing Radix Tooltip + registry architecture.

## Implementation Phases

### Phase 1 — Infrastructure (blocking; all subsequent phases depend on this)

1. Install `@radix-ui/react-hover-card` and `@radix-ui/react-popover` in `frontend/package.json`
2. Create `hover-card.tsx` in `components/ui/` — Radix HoverCard wrapper with `openDelay={300}`, `closeDelay={150}`, fade+slide animation, `prefers-reduced-motion` support, JSX-rich `content` prop
3. Create `popover.tsx` in `components/ui/` — Radix Popover wrapper with focus trap, Escape-to-close, outside-click-to-close, focus-return-to-trigger, `aria-haspopup`/`aria-expanded`
4. Extend `tooltip-content.ts` registry with all missing keys identified in Phase 2 audits (~40+ new entries)
5. Unit tests for both new primitives: `hover-card.test.tsx`, `popover.test.tsx`
6. Extend `test-utils.tsx` to wrap with HoverCard support if needed

### Phase 2 — Component-by-Component Gap Fill (parallel once Phase 1 complete)

- **2.1 Pipeline Builder**: Drag handle tooltips, clone button tooltip, complexity badge tooltip, AgentNode hover card, ModelSelector popover migration, SavedWorkflowsList title= → Tooltip
- **2.2 Agents Page**: AgentCard icon/expand tooltips, AgentCard hover card, ToolsEditor move/remove tooltips, BulkModelUpdateDialog scope tooltip
- **2.3 Chat Interface**: VoiceInputButton stop tooltip, MentionAutocomplete agent descriptions, CommandAutocomplete descriptions, @agent-name hover card
- **2.4 Board / Projects**: AddAgentPopover migration, AgentPresetSelector migration, filter/sort tooltips, column header tooltips, IssueCard hover card
- **2.5 Tools Page**: ToolCard re-sync tooltip, RepoConfigPanel field help, GitHubMcpConfigGenerator copy tooltip, search input tooltip
- **2.6 Settings Page**: Temperature slider tooltip, provider toggle tooltip, theme toggle title= migration, Signal QR tooltip, destructive reset warnings
- **2.7 Sidebar**: Replace all `title=` on icon-only nav items with `<Tooltip contentKey="nav.*">`, collapse/expand toggle tooltip

### Phase 3 — Hover Styling Consistency

- Establish uniform Tailwind hover state tokens (`hover:ring-1 hover:ring-border`, `hover:bg-accent/50`)
- Apply to all interactive card surfaces: AgentCard, IssueCard, ToolCard, ChoreCard, AgentNode, StageCard
- Drag handle affordance: `cursor-grab` + `opacity-0 group-hover:opacity-100` for all draggable items
- Drop zone feedback: `isOver` state drives ring/background shift on `@dnd-kit` drop targets

### Phase 4 — Popover Migration Audit

- Grep for remaining manual `useState` + absolute-positioned overlay patterns
- Migrate each to `popover.tsx`
- Verify: Escape closes, focus returns, `aria-haspopup`/`aria-expanded` correct

### Phase 5 — Registry Audit & Consistency

- Prune orphaned keys in `tooltip-content.ts` (no `contentKey=` usage)
- Eliminate remaining `title=` on interactive elements
- Terminology pass: tooltip copy must match UI labels exactly

### Phase 6 — Accessibility & Motion

- `aria-label` audit: every icon-only button must have `aria-label` matching tooltip text
- `prefers-reduced-motion`: HoverCard and Popover animations use existing Celestial motion tokens
- Keyboard reachability: all tooltip triggers Tab-reachable; all popovers focus-trapped

### Phase 7 — Tests & Verification

- Unit tests per migrated popover + new hover card integration
- Accessibility scanning: zero axe violations on all pages
- Manual smoke walk verification
