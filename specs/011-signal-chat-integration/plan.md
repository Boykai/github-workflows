# Implementation Plan: Signal Messaging Integration

## Summary

Add Signal messaging integration to the existing chat UX, allowing users to link their Signal account via QR code, receive app chat messages (assistant/system) as Signal notifications, and send messages from Signal into the app's AI-powered chat workflow. Includes notification preference controls, encrypted phone number storage, and exponential-backoff retry for outbound delivery.

## Technical Context

**Language/Version**: Python 3.11+ (backend), TypeScript ~5.4 (frontend)
**Primary Dependencies**: FastAPI, Pydantic 2, httpx, aiosqlite (backend); React 18, TanStack Query, Vite (frontend)
**Storage**: SQLite via aiosqlite (existing pattern — new migration for `signal_connections` and `signal_messages` tables)
**Testing**: pytest + pytest-asyncio (backend), Vitest + Testing Library (frontend), Playwright (e2e)
**Target Platform**: Linux server (Docker), browser SPA
**Project Type**: Web application (backend + frontend)
**Performance Goals**: Outbound Signal delivery within 30 seconds; inbound message processing within 30 seconds
**Constraints**: Signal integration must not degrade existing chat functionality; phone numbers encrypted at rest (Fernet, existing `EncryptionService`); PII deleted on disconnect
**Scale/Scope**: Per-user Signal connection; best-effort delivery channel; text-only for initial release
