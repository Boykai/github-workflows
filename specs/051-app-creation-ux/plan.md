# Implementation Plan: Debug & Fix Apps Page — New App Creation UX

**Branch**: `051-app-creation-ux` | **Date**: 2026-03-18 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/051-app-creation-ux/spec.md`

## Summary

The Solune "Create New App" flow creates a GitHub repository and commits scaffold template files but has three critical gaps: (1) template file failures are silently skipped, (2) no Parent Issue is created to kickstart the Agent Pipeline, and (3) the frontend lacks pipeline selection and post-creation feedback. This plan addresses all three by hardening template file error propagation, wiring existing orchestration primitives (`create_issue`, `create_all_sub_issues`, `ensure_polling_started`) into the app creation flow, adding database columns for parent issue tracking, and enhancing the frontend with pipeline selection, full warning display, and structured creation feedback.

## Technical Context

**Language/Version**: Python ≥3.12 (backend), TypeScript/React 19 (frontend)
**Primary Dependencies**: FastAPI ≥0.135, Pydantic ≥2.12, aiosqlite ≥0.22, githubkit ≥0.14.6 (backend); React 19.2, TanStack React Query 5.90, Vite 7.3, Radix UI (frontend)
**Storage**: SQLite via aiosqlite (persistent module-level connection, `init_database()` / `get_db()`)
**Testing**: pytest + pytest-asyncio (backend), Vitest 4.0 + @testing-library/react (frontend)
**Target Platform**: Linux server (Docker Compose), browser SPA
**Project Type**: Web application (backend + frontend)
**Performance Goals**: App creation completes within 30 seconds; branch poll tolerates up to 15 seconds of GitHub delay; pipeline agent assigned within 60 seconds
**Constraints**: Best-effort parent issue creation (non-blocking); backward compatibility for existing apps without parent issues
**Scale/Scope**: Single-user Solune instance, ~10–50 apps per deployment
