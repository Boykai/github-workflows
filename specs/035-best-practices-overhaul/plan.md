# Implementation Plan: Codebase Improvement Plan — Modern Best Practices Overhaul

**Branch**: `035-best-practices-overhaul` | **Date**: 2026-03-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/035-best-practices-overhaul/spec.md`

## Summary

Full-stack audit and overhaul of the existing web application across 7 phases: (1) persist volatile in-memory pipeline and chat state to SQLite with write-through caching and async-safe locking, (2) decompose the 5 highest cyclomatic-complexity functions into focused handlers/sub-hooks, (3) consolidate 8 duplicate repository-resolution paths into one canonical function, adopt existing error infrastructure, and add Pydantic models for all API inputs, (4) replace module-level singletons with FastAPI lifespan + `Depends()` and break circular imports via a shared interfaces module, (5) harden CORS, add CSP middleware, require explicit admin designation, and scope sessions to users, (6) enforce structured error logging, add frontend telemetry, create integration tests, tighten assertions, and remove committed build artifacts, (7) restructure frontend hooks to return grouped objects and standardize retry logic on `tenacity`. Phase 1 blocks all others; Phases 2–5 run in parallel; Phases 6–7 are independent.

## Technical Context

**Language/Version**: Python 3.13 (backend), TypeScript / ES2022 (frontend)
**Primary Dependencies**: FastAPI 0.135+, Pydantic 2.12+, aiosqlite, githubkit, React 19, TanStack Query v5, Vite 7.3, Tailwind v4, Zod v4
**Storage**: SQLite via aiosqlite (async), with write-through in-memory `BoundedDict` cache
**Testing**: pytest + pytest-asyncio (backend), Vitest + happy-dom (frontend unit), Playwright (e2e)
**Target Platform**: Linux container (Docker), modern browsers
**Project Type**: Web application (FastAPI backend + React SPA frontend)
**Performance Goals**: No regression from current baseline; write-through cache adds ≤5 ms p95 per state mutation
**Constraints**: Zero data loss on container restart; all functions ≤25 CC; zero bare `except Exception` blocks
**Scale/Scope**: ~465 files, ~4,567 functions, single-instance deployment with SQLite
